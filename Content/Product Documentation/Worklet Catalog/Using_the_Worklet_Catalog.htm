Using the Worklet Catalog

The Worklet Catalog allows you to immediately use a verified Worklet for your purposes. You can also create a new policy based on an existing worklet.

Accessing the Worklet Catalog

You can access the catalog from the Automate → Worklet Catalog page.

Worklet Catalog Table

Description

Worklet Name

Name assigned by Automox

Type

This can be a workstation or server

OS

This is the OS type that the worklet is intended for.

Categories

Worklets are organized into categories: Data Collection & Auditing, Maintenance Tasks, Peripherals, Personalization, Security, Software Lifecycle, System Preference

Last Updated

Time and date when the worklet was last updated

Actions

Possible actions:

  • Select View Details to find out what the verified worklet is used for.
  • Select Create Policy to customize the existing worklet for your own purposes.

Search the Worklet Catalog

From the Workout Catalog page, you can use the search bar to search for a worklet by a variety of terms.

Using the filter panel

The filter panel is made up of various options to fine-tune your search. When you select any of the individual filters, a number in parentheses shows how many are selected. You can clear selections individually, or select Clear All. The panel can also be collapsed. These settings are saved in the URL. When you navigate to other pages and return to the page, your settings are maintained.

You can select from the following options:

  • OS
  • Type
  • Categories
  • What’s New

Using the search bar

You can enter partial names and as many search items as you want. For example:

  • Name
  • Description
  • Operating system
  • Keyword

  1. Search by Name values supports fuzzy search, which takes misspellings into consideration. Other search terms (such as description, OS, or keyword) use a wildcard match.
  2. Click the column heading to sort by that type of data. You can now also turn off a sort when a column is sorted a third time, which reverts results back to sorted by relevancy.

The results include any matching data, or close to matching data. Search results are returned by relevance to the search term.

You can also refer to our website to search through the growing list of worklets: Browse Worklets

Viewing Details

There are two ways to view the details of a verified Automox worklet.

  1. Click the Worklet Name in the table. The details view opens.
  2. You can also open the Actions menu for the worklet you want to view details for. Click View Details.

In the Worklet Details window, select Code Preview to open the Evaluation and Remediation Code details for the selected worklet.

You can scroll through the code to view the details.

  • To edit the code presented in a worklet from the catalog, you must first add it to a policy. Click Create Policy.
  • The Create Worklet page opens and you can make the desired changes, such as:
    • configure device targeting
    • edit code
    • set a schedule
    • configure user notifications
